## 터미널과 첫 명령어 `ls`

이번 레슨부터 오른쪽 터미널을 직접 사용합니다. 터미널은 키보드로 명령을 입력해 컴퓨터에게 일을 시키는 창입니다.

가장 기본이 되는 명령어부터 쳐봅시다. `ls`(list의 줄임말)는 현재 폴더에 어떤 파일이 있는지 보여줍니다.

```bash
ls
```

아무것도 나오지 않아도 정상입니다. 지금 이 폴더가 비어 있다는 뜻이니까요. 이 "비어 있음"을 기억해 두세요.

## 저장소 만들기

Git으로 프로젝트를 관리하려면 먼저 **저장소**(Repository)를 만들어야 합니다. 명령어는 `git init`(initialize, 초기화):

```bash
git init
```

실행하면 이런 메시지가 나옵니다:

```
Initialized empty Git repository in /.git/
```

첫 번째 Git 저장소가 만들어졌습니다.

## 무엇이 달라졌을까?

다시 `ls`를 쳐보면 여전히 아무것도 없어 보입니다. 하지만 숨김 파일까지 보여주는 `-a` 옵션을 붙이면:

```bash
ls -a
```

`.git`이라는 폴더가 새로 생긴 것을 볼 수 있습니다. 이름이 `.`으로 시작하는 파일은 숨김 처리되어 평소에는 보이지 않습니다.

> `ls -al`처럼 `-l`을 함께 붙이면 파일 종류와 크기까지 자세히 표시됩니다.

## .git 디렉토리란?

`git init`이 만든 `.git` 폴더가 바로 Git 저장소의 심장입니다. 이 안에 저장되는 것:

- 모든 커밋 이력
- 브랜치 정보
- 설정 파일

> **주의**: `.git` 폴더를 직접 수정하거나 삭제하면 저장소가 손상될 수 있습니다. 이 폴더는 Git이 관리하도록 두세요.

## GitHub 화면 직접 둘러보기

이번 레슨은 읽기만 하지 않습니다. 새 탭에서 GitHub를 열고, 아래 항목을 하나씩 직접 확인해보세요. 둘러보기는 로그인 없이도 됩니다.

먼저 이 저장소를 여세요. 여러분이 지금 쓰고 있는 이 튜토리얼 사이트도 GitHub에 올라가 있습니다:

> 👉 [github.com/0-Chan/git-github-101](https://github.com/0-Chan/git-github-101)

## 저장소 첫 화면 (Code 탭)

저장소에 들어가면 보이는 화면입니다. 이것만 찾아보세요:

- **파일 목록**: 프로젝트의 폴더와 파일. 클릭하면 코드를 바로 읽을 수 있습니다.
- **최근 커밋 한 줄**: 파일 목록 위에 마지막 커밋 메시지와 시간이 보입니다.
- **커밋 히스토리**: 오른쪽 위 시계 아이콘(커밋 개수)을 클릭해보세요. 터미널에서 친 `git log`가 여기서는 이렇게 보입니다.
- **브랜치 드롭다운**: 왼쪽 위 `main` 버튼. 다른 브랜치로 화면을 전환할 수 있습니다.

## README

파일 목록 아래에 보이는 긴 설명 문서가 `README.md`입니다. 저장소의 얼굴이라, 마크다운 파일 하나가 프로젝트 소개 페이지 역할을 합니다.

## Star, Fork, Watch

저장소 오른쪽 위의 버튼 세 개:

| 버튼 | 역할 |
| --- | --- |
| **Star** | 즐겨찾기. 좋은 프로젝트에 누르는 좋아요입니다 |
| **Fork** | 남의 저장소를 내 계정으로 통째로 복사 |
| **Watch** | 저장소의 새 소식을 알림으로 구독 |

**Fork** 버튼의 위치를 기억해두세요. 2회차 실전 미션이 여기서 시작됩니다.

## Issues

이번에는 활동이 많은 대형 저장소로 가봅시다:

> 👉 [github.com/facebook/react](https://github.com/facebook/react)

**Issues** 탭을 열어보세요. 버그 신고, 기능 제안, 할 일이 목록으로 관리됩니다. 항목마다 붙은 색깔 태그(라벨)와 담당자를 확인해보세요. 오픈소스 팀은 이슈로 대화합니다.

## Pull requests

**Pull requests** 탭에서 열려 있는 PR을 아무거나 하나 클릭해보세요:

- **Conversation 탭**: 변경 이유 설명과 리뷰 대화가 오갑니다.
- **Files changed 탭**: 바뀐 코드가 초록(추가)과 빨강(삭제)으로 표시되고, 코드 줄마다 리뷰 댓글이 달립니다.

레슨 11에서 배운 코드 리뷰 워크플로우의 실물이 바로 이 화면입니다.

## Actions

**Actions** 탭에는 커밋마다 자동으로 실행된 작업 기록이 쌓입니다. 커밋 옆에 붙는 초록 체크 ✓ 와 빨간 X의 정체가 이것입니다. 코드가 올라올 때마다 테스트를 자동으로 돌린 결과죠. 3회차에서 직접 만들어봅니다.

## 프로필과 잔디

GitHub 계정이 있다면 내 프로필(`github.com/내아이디`)을 열어보세요. 초록 칸이 채워지는 **contribution graph**, 일명 잔디가 보입니다. 커밋과 PR 활동이 하루 단위로 기록되는 곳입니다.

지금은 비어 있어도 괜찮습니다. 2회차 미션을 마치면 첫 칸이 채워집니다.

## 다음 시간 예고: 실제 GitHub에서 해보기

지금까지의 실습은 브라우저 안의 연습용 터미널이었습니다. 2회차에는 오늘 배운 흐름을 실제 GitHub에서 그대로 실행합니다.

- **Git 심층 탐구**: 지금까지 스쳐 지나간 용어들의 정체를 밝힙니다. `HEAD`는 무엇을 가리키는지, `origin`과 `upstream`은 왜 그런 이름인지
- **fork와 clone**: 다른 사람의 저장소를 내 계정으로 복사하고, 내 컴퓨터로 내려받기
- **진짜 push**: 내 커밋을 실제 원격 저장소에 올리기
- **첫 Pull Request**: PR을 직접 만들고 리뷰를 거쳐 머지까지
- 이 과정을 마치면 내 GitHub 계정에 **첫 기여 기록**이 남습니다

> **준비물**: GitHub 계정이 필요합니다. 아직 없다면 [github.com](https://github.com)에서 미리 가입해 두세요.

축하합니다! 1일차 과정을 모두 완료했습니다. 버전 관리의 기초를 갖추었으니, 다음 시간에는 실전으로 갑니다.
